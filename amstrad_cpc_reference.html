CPC 464 / 664 / 6128

► AMSTRAD COLOUR PERSONAL COMPUTER · 1984–1990

The Amstrad CPC range brought a complete 8-bit computing system — monitor included — to the mass market at an aggressive price point. Running a Z80A at 4 MHz with the proprietary Gate Array 40010 for video, three-channel AY sound, and Locomotive BASIC in ROM, the CPC sold approximately 3 million units across Europe and built one of the era's most vibrant software libraries.

Overview & History

Designed by Amstrad Consumer Electronics (Alan Sugar) and launched in the UK in June 1984, the CPC was a deliberate response to the fragmented British home computer market. By bundling a monitor and built-in storage, Amstrad eliminated the extra costs that plagued ZX Spectrum and Commodore 64 buyers.

1984
CPC 464 launches in the UK at £199 (green monitor) / £299 (colour). Simultaneous European rollout in Germany, France, Spain, Australia.
1985
CPC 664 introduced — built-in 3" floppy replacing the tape deck. CPC 6128 follows quickly: 128KB RAM, CP/M Plus included. CPC 664 discontinued within months.
1986
Amstrad acquires Sinclair brand. CPC continues alongside the Spectrum range.
1990
CPC Plus range (464+, 6128+) and GX4000 games console released with hardware sprite ASIC. Classic CPC range discontinued.

Screen RAM Address Calculation

The CPC uses an interleaved raster layout — consecutive bytes do NOT go to consecutive scan lines. Character rows are interleaved across 8 scan lines.

; Screen address for (X_byte, Y) with base &C000:
; Address = &C000 + (char_row × 80) + (scan_line × 2048) + X_byte
; Where: char_row = Y / 8,  scan_line = Y MOD 8

SCREEN_ADDR:
    LD  A, L          ; L = Y coordinate
    AND &07          ; Scan line within char row
    RRCA              ; × 256 high byte
    RRCA
    RRCA
    OR  &C0          ; Add base &C000
    LD  H, A
    LD  A, L
    AND &F8          ; Char row × 8
    RRCA
    RRCA
    RRCA              ; ÷ 8 then × 80 = × 10
    ADD A, 80        ; Actually multiply properly...
    LD  L, A          ; HL = screen address
Graphics Coordinate SystemBASIC graphics always use a 640×400 coordinate system regardless of MODE. In MODE 0 each physical pixel = 4×2 units; in MODE 1 = 2×2 units; in MODE 2 = 1×2 units.

Interrupts

The Gate Array fires an interrupt every 52 scan lines = 6× per frame at 50 Hz = 300 Hz. The firmware uses this for the system clock, keyboard scanning, SOUND queue management, and AFTER/EVERY BASIC timers.

Adding a Timer Interrupt (Firmware)

; KL ADD TICKER at &BC0E — fires custom routine at regular intervals
; Ticker block: 2 bytes countdown, 2 bytes reload, 2 bytes routine address

ticker_block:
    DW  6              ; Countdown (decremented each interrupt)
    DW  6              ; Reload value
    DW  my_routine     ; Routine address (called at count=0)

install_ticker:
    LD  HL, ticker_block
    CALL &BC0E          ; KL ADD TICKER — fires my_routine at 50Hz
    RET

my_routine:          ; Called every 6 Gate Array interrupts (50Hz)
    PUSH AF            ; Must preserve ALL registers
    ; ... your per-frame code here ...
    POP  AF
    RET

wait_vsync:          ; Wait for vertical sync
    IN  A, (&F5)       ; Read PPI Port B
    AND &80            ; Bit 7 = VSYNC
    JR  Z, wait_vsync
    RET

Reading Keyboard in Assembly

; Read keyboard row and check for keypress
; PPI Port C lower nibble → row select
; AY Port A (R14) → column bits (0 = pressed)

KM_TEST_ROW:    ; Input: A = row (0-9)
    IN  B, (&F6)    ; Read current Port C
    AND &F0         ; Preserve upper bits
    OR  B           ; Mix in row number (A must be cleared first)
    OUT (&F6), A    ; Set keyboard row
    ; Now read AY register 14 via PPI
    LD  A, 14
    OUT (&F4), A    ; Select AY R14
    LD  A, &40     ; BC1=1, BDIR=0 → Read
    OUT (&F6), A
    IN  A, (&F4)   ; Read column bits
    CPL             ; Invert (1 = pressed)
    RET

Hello World

ORG &4000

    LD  HL, hello_str   ; Point to string

Block Move (LDIR)

LD  HL, source      ; Source address
LD  DE, dest        ; Destination address
LD  BC, 4096        ; Byte count
LDIR                ; Copy 4KB — hardware accelerated!

Hardware Scroll

SCROLL_RIGHT:
    LD  BC, &BC0D       ; Select CRTC register 13 (offset low)
    OUT (C), B
    LD  A, (scroll_lo)
    INC A
    LD  (scroll_lo), A
    LD  B, &BD
    OUT (C), A          ; Write new offset (2 pixels right in MODE 1)
    RET
scroll_lo: DEFB 0

; C major chord: C(239), E(190), G(159) in octave 4
PLAY_CHORD:

🚀 Getting Started with WinAPE (Recommended — Windows)

WinAPE is the most accurate and feature-complete CPC emulator. Includes a built-in assembler and debugger.

Download & Install

Get the latest version from winape.net/downloads.jsp. Extract the zip to any folder — no installer needed. Run WinAPE.exe.

Choose Your Model

Go to Hardware → CPC Type. Select CPC 464 for tape software, CPC 6128 for disk software and maximum RAM. The 664 is rare — skip unless specifically needed.

Load a Tape (.CDT / .CSW)

Go to Tape → Insert Tape → select your .cdt file. At the BASIC prompt type RUN"" and press Enter. The tape will auto-play.

Load a Disk Image (.DSK)

Go to Drive A: → Insert Disk Image → select your .dsk file. At the BASIC prompt type CAT to list contents, then RUN"FILENAME" to run a program.

Use the Built-in Assembler

Press F8 to open the assembler/debugger. You can write Z80 assembly directly, set breakpoints, and inspect registers. Essential for demoscene or game development work.

🌐 No-Install Option: CPC-EMU (Browser)

Perfect for quickly testing software. Open cpcemu.arnoldemu.com in any browser. Drag and drop a .dsk or .cdt file onto the screen to load it instantly.
