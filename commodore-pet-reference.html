📋 Step-by-Step: Your First Day With a PET

1

Identify Your Machine

Check the label on the front or back. The model number encodes screen columns and RAM — 4032 = 40 column, 32 KB. 8032 = 80 column, 32 KB. Use the wizard above or visit the Models & History tab for the full table. Note whether you have a 9" or 12" CRT — this tells you if you have the older non-CRTC board or the newer CRTC board.

2

First Power-On Check

Power on and watch carefully. A healthy PET shows a cursor and the BASIC version string within 2 seconds. Random garbage characters = failing DRAM (check -5V rail first). Black screen = ROM/RAM fault (ROM/RAM board fixes ~80% of these). Cursor but no keyboard = PIA 1 (6520) fault or dead keyboard membrane. See Board ID & Schematics for the full fault guide.

3

Check Your BASIC Version

Type: PRINT PEEK(224) — response tells you your BASIC version: 0 = BASIC 1.0 (very buggy, upgrade recommended), 96 = BASIC 2.0 (stable), 176 = BASIC 4.0 (adds disk commands, best). If you have BASIC 1.0, the ROM/RAM board upgrade is the most impactful thing you can do.

4

Set Graphics Mode & Try BASIC

Type POKE 59468,12 to ensure graphics mode is active. Then try: PRINT "HELLO FROM 1977!" — you should see your text. Try 10 PRINT "PET";:GOTO 10 then RUN for the classic PETSCII demo. Press STOP to halt it.

5

Load Your First Program

With SD2PET (IEEE-488 device 8): LOAD "$",8 then LIST to see the directory, then LOAD "GAME",8 and RUN. With cassette: LOAD (no quotes) and press PLAY on the tape. In emulator: drag a .prg file onto the masswerk emulator window — it loads automatically.

6

Explore & Expand

The BASIC tab has syntax reference and code examples. The Homebrew tab lists the best modern games with download links. The Hardware Today tab covers what to buy to make your machine more reliable and functional. The Shops & Blogs tab has the repair resources you'll need for any hardware issues.

Board ID & Schematics

The PET used several distinct motherboard revisions across its production run. Knowing which board you have is essential before attempting repairs — especially to avoid the Killer POKE on CRTC boards. This section helps you identify your revision visually and links to the key schematics.

☠️
CRTC WARNING: Boards 3, 4 (universal) and all 8000-series boards have a MOS 6545 CRTC chip at $E880. The old speed-up POKE POKE 59458,62 writes to this register and can permanently damage the monitor. Only use it on Board 1 and Board 2 (9" non-CRTC) machines.

The Machine That Started It All

The Commodore PET (Personal Electronic Transactor) was Commodore's first successful mass-market personal computer, launched at the January 1977 Winter CES alongside the Apple II and TRS-80 — the "1977 Trinity" that launched the home computing revolution.

Designed primarily by Chuck Peddle (the driving force behind the MOS 6502 CPU itself), the PET came as a fully integrated all-in-one machine with built-in monitor, keyboard, and cassette drive — a complete system straight out of the box, which was genuinely revolutionary for the time.

Unlike its contemporaries, the PET was targeted squarely at education and business as much as home use. Its rugged all-metal chassis, IEEE-488 interface for external peripherals, and robust BASIC made it extremely popular in UK schools throughout the late 1970s and into the 1980s. Many a British programmer cut their teeth on a PET in the classroom.

The PET spawned an entire lineage of Commodore computers — the VIC-20, C64, and C128 all owe their DNA to the engineering work first done on the PET family.

🎓

The School Computer

The PET dominated UK schools in the early 1980s. Its tough metal construction, simple BASIC, and built-in everything made it ideal for education. Generations of British programmers wrote their first lines of code on a PET 3032 or 4032.

💼

Business Machine

The 8032 CBM series with its 80-column display targeted business users directly. Running business software like VisiCalc, WordPro, and database tools, PETs could be found in offices throughout the late 1970s and early 1980s.

🔬

IEEE-488 Power

The IEEE-488 (GPIB) interface let the PET control lab instruments and dual-drive floppy systems (4040, 8050, 8250). This made it exceptionally capable for scientific and industrial environments where most home computers couldn't compete.

⚠️

Famous Easter Egg

On models with BASIC 2.0+, type WAIT 6502,X — the screen fills with "MICROSOFT!". Bill Gates himself reportedly inserted this to assert copyright after a dispute with Jack Tramiel. It doesn't work on BASIC 1.0.

☠️

The Killer POKE

On CRTC-based PETs (4000/8000 series), the old speed-up trick POKE 59458,62 writes to the 6545 CRTC register instead, corrupting the refresh rate. This can permanently damage the monitor. Never use it on CRTC models!

🎮

PETSCII Graphics

The PET's character set included 64 block and line-drawing graphics characters, enabling surprisingly good "text mode" games and displays. The PETSCII standard (PET Standard Code of Information Interchange) became the basis for all later Commodore machines.

💡
PET vs CBM: The "PET" name only officially applies to the original 2001 series and some 4000-series machines. Due to a trademark dispute with Philips in Europe, models sold there used "CBM" (Commodore Business Machines). In North America, CBM was used for the business-oriented models. Collectors and enthusiasts still call them all "PETs" regardless.

Models & History

1977
PET 2001 (Original)
The original machine. Chiclet keyboard, built-in cassette beneath the keyboard, 9" CRT, 4 KB or 8 KB RAM. BASIC 1.0 was notoriously buggy (floating point errors, missing commands). The tape drive was unreliable. Despite flaws, it sold out instantly.
BASIC 1.0 4–8 KB RAM Chiclet KB Buggy!
1978
PET 2001-N (New ROM)
Major revision. Full-travel keyboard replaced the chiclet, external cassette port added (tape now external), BASIC 2.0 fixed most bugs. Available in 8, 16, and 32 KB RAM. This is the model most people picture when they think "PET 2001".
BASIC 2.0 8–32 KB RAM Normal KB
1979
CBM 3000 Series (Europe)
European rebrand due to Philips' PET trademark. CBM 3008, 3016, 3032. Functionally identical to the 2001-N series with BASIC 2.0/2.3. The 3032 (40-col, 32 KB) became extremely popular in British schools.
BASIC 2.0/2.3 EU Only 9" CRT
1980
PET/CBM 4000 Series
Major upgrade with BASIC 4.0, adding disk commands (DLOAD, DSAVE, SCRATCH, etc.). Two sub-families: "thin" 9" models and "fat" 12" CRTC models. The 4032 was the most popular variant. Memory sockets punched out on 4008/4016 to prevent cheap RAM upgrades!
BASIC 4.0 9" or 12" CRT 6545 CRTC (12")
1981
CBM 8000 Series (80-col)
80-column display for business use. The 8032 (80-col, 32 KB) used the new universal motherboard with 6545 CRTC. Business keyboard standard. The piezo speaker made its debut. 8096 added 64 KB expansion RAM via paged memory.
BASIC 4.0 80-col Piezo Beeper
1982
SuperPET 9000 (MMF 9000)
The ultimate PET — co-developed with the University of Waterloo, Canada. Added a Motorola 6809 CPU alongside the 6502, enabling CP/M-like operating systems (WATERLOO BASIC, COBOL, FORTRAN, APL, Pascal). Two add-on boards fit inside the standard 8032 case.
6502 + 6809 Multi-language Expensive!
1984
CBM 8032-SK / 8296 "Porsche PET"
The final PET revival. Rounded plastic case with swivel monitor and separate keyboard (SK = SoftKey). The 8296 had 128 KB RAM with paged banking. Sold mainly in Europe. Visually striking machines nicknamed "Porsche PETs" for their sleek design.
128 KB (8296) Separate KB EU Only

🧠 CPU

⚙️

MOS Technology 6502 — CPU All Models

  • Manufacturer MOS Technology (later Rockwell, WDC)
  • Clock Speed 1 MHz (PET runs at exactly 1.0 MHz)
  • Data Bus 8-bit
  • Address Bus 16-bit (64 KB address space)
  • Transistors ~3,510 (NMOS process)
  • Package 40-pin DIP
  • Designer Chuck Peddle, Bill Mensch — MOS Tech 1975
  • Variants on PET 6502, 6502A (1 MHz), later W65C02S

The 6502 has 6 internal registers:

A — Accumulator (8-bit) Arithmetic & logic operations target X — Index Register X (8-bit) Loop counters, address offsets Y — Index Register Y (8-bit) Loop counters, address offsets S — Stack Pointer (8-bit) Points into $0100–$01FF (grows downward) PC — Program Counter (16-bit) Address of next instruction to execute P — Processor Status Flags (8-bit) Bit: 7 6 5 4 3 2 1 0 N V - B D I Z C │ │ │ │ │ │ └── Carry │ │ │ │ │ └───── Zero │ │ │ │ └──────── IRQ Disable │ │ │ └─────────── Decimal (BCD) │ │ └────────────── Break (BRK instr) │ └──────────────────── Overflow └─────────────────────── Negative

13 Addressing Modes:

Implied CLC ; no operand Accumulator ASL A ; operand is A Immediate LDA #$42 ; operand is literal value Zero Page LDA $42 ; 2-byte, fast — address $0000–$00FF Zero Page,X LDA $42,X ; zero page + X offset Zero Page,Y LDA $42,Y ; zero page + Y offset Absolute LDA $8000 ; 3-byte — any 16-bit address Absolute,X LDA $8000,X ; absolute + X (crosses pages = +1 cycle) Absolute,Y LDA $8000,Y ; absolute + Y (Indirect,X) LDA ($42,X) ; pointer from zero page + X (Indirect),Y LDA ($42),Y ; dereference zero page, then + Y Indirect JMP ($8000) ; for JMP only — 16-bit pointer Relative BEQ $05 ; branch ±127 bytes from next PC

Interrupt handling: IRQ (maskable, triggered by hardware) pushes PC+P to stack, loads vector from $FFFE/$FFFF. NMI (non-maskable) uses $FFFA/$FFFB. RESET loads from $FFFC/$FFFD. On the PET, IRQ fires 60 times/second from the video circuit — the kernal uses this for the real-time clock and keyboard scan.

💾 RAM

🟩

MOS 6550 — Static RAM Board 1 Only

  • Type SRAM (Static RAM)
  • Capacity 1 KB per chip (1024 × 4 bit)
  • Chips on board 8× = 4 KB, 16× = 8 KB
  • Package 22-pin DIP
  • Supply Voltage +5V only (single rail — simple!)
  • Cycle Time ~450 ns
  • Reliability Excellent — no refresh needed
  • Models PET 2001 (4K and 8K) only
💡
The 6550 was expensive — it's the main reason early 2001s had only 4–8 KB. Commodore switched to cheaper 4116 DRAM for all later models. 6550s rarely fail and don't need the -5V rail that causes so many 4116 problems.

🟨

MOS/Intel 4116 — Dynamic RAM Boards 2–5

  • Type DRAM (Dynamic RAM — needs refresh)
  • Capacity 16 Kbit per chip (16,384 × 1 bit)
  • Chips for 32 KB 16× 4116 (8 for data, 8 for parity)
  • Package 16-pin DIP
  • Supply Voltages THREE rails: +12V, +5V, −5V
  • Refresh Every 2 ms, 128-cycle RAS-only refresh
  • Cycle Time ~150–200 ns
  • Reliability Notoriously poor — main failure point
☠️
The −5V rail is critical. If it fails or drifts, ALL 4116s produce garbled output simultaneously — looks like random screen garbage. Always measure −5V first before replacing chips. A missing −5V is a PSU or capacitor fault, not a RAM fault.

🖥️ Video Hardware

🎞️

Hardware Video Circuit Boards 1 & 2 — 9" Models

  • Type Discrete logic — no dedicated video chip
  • Clock Source 16 MHz crystal ÷ 16 = 1 MHz CPU, ÷ 8 = dot clock
  • Video RAM 1 KB at $8000, read by hardware at 1 MHz
  • Character ROM 6540 addressed directly by video counter
  • Sync Composite sync, internal 9" CRT only
  • Pixel Clock ~8 MHz dot clock
  • Screen 40 columns × 25 rows, 9" green phosphor
  • Bandwidth share CPU and video share bus — no DMA needed

On early PETs the video circuit is built from 74-series TTL logic chips — counters, multiplexers, and shift registers — rather than a dedicated chip. The circuit alternates video RAM access with CPU access on alternate clock cycles, so both can use the same memory bus without conflict. This is why the early PET doesn't need a DMA controller or a "bad line" stall like the C64.

⚡ Clock & Power Supply

🔄

Crystal Oscillator & Clock Circuit

  • Crystal Frequency 16.000 MHz (NTSC) / 16.000 MHz (PAL)
  • CPU Clock 16 MHz ÷ 16 = 1.000 MHz
  • Dot Clock 16 MHz ÷ 2 = 8 MHz (for video pixel output)
  • IRQ Rate Derived from video VSYNC = 60 Hz (NTSC) or 50 Hz (PAL)
  • Divider Circuit 74LS163 binary counter chain
  • Clock Driver 74LS04 inverter / clock buffer to 6502 Φ2
  • PAL vs NTSC Different crystal on PAL machines for 50 Hz timing

The 60 Hz IRQ is the heartbeat of the PET — it drives the software real-time clock (TI$ variable), debounces the keyboard, and updates the cursor blink. PAL machines run at 50 Hz which slightly slows the software clock unless the kernal compensates.

🖥️ VICE / XPET — First-Time Setup Walkthrough

VICE is the go-to desktop PET emulator but the initial setup trips up many people. Follow these steps to go from download to running your first .d64 or .prg file.

1

Download VICE

Go to vice-emu.sourceforge.io → Downloads. Pick the right build: Windows = the .zip with "win64" in the name. macOS = use Homebrew (brew install vice) or the .dmg. Linux = your distro's package manager (sudo apt install vice). Extract to a permanent location — don't run from Downloads.

2

Launch XPET (not x64 or x128)

The VICE suite has multiple executables. xpet (or xpet.exe) is the PET emulator specifically. On Windows it's in the root of the extracted folder. Do not double-click x64.exe — that's the C64 emulator. First launch will show a PET 2001 by default.

3

Select Your PET Model

Go to Settings → PET Model Settings. Choose from the dropdown:
PET 4032 — 40-col, 32 KB, BASIC 4.0 (most compatible, recommended default)
PET 8032 — 80-col, 32 KB, BASIC 4.0 (for 80-column software)
PET 2001-8N — original, 8 KB, BASIC 1.0 (historic accuracy only)
After selecting, go to Settings → Save Settings or it will reset next launch.

4

Attach a Disk Image (.d64)

Go to File → Attach Disk Image → Drive 8. Navigate to your .d64 file and click Attach. The drive is now device 8. In the emulator type:

LOAD "$",8 ← Load directory LIST ← Show disk contents LOAD "GAME",8 ← Load a specific program RUN

Or for BASIC 4.0 shorthand: DIRECTORY then DLOAD "GAME"

5

Load a .prg File Directly

For standalone .prg files (no disk image needed): File → Smart Attach — VICE detects the file type and loads it automatically. Alternatively drag-and-drop a .prg onto the VICE window. After attaching, type RUN in the emulator.

6

💡 Pro Tips for Running Software

📐
40 vs. 80 Columns: Most games are written for 40-column machines (2001, 3032, 4032). On an 8032, many games appear in a small box on the left side. You may need a "40-column emulator" ROM or the correct hardware jumper setting to run them full-screen.
⌨️
PETSCII vs. Graphics Mode: If a game looks like a mess of symbols, check your character mode. Toggle with:
POKE 59468,12 — Graphics mode (default, needed for most games)
POKE 59468,14 — Lowercase mode (for text-heavy programs)
💾
SD2PET — The Gold Standard: If you're on real hardware, the SD2PET from TFW8b is the best way to load .prg files. It appears as a standard IEEE-488 drive (device 8) — just drop files on an SD card and use normal LOAD commands. No cassette fiddling required.
🖥️
Using VICE / XPET: Select your target model before loading (Model menu → PET 4032 is the safest default for most modern homebrew). Load .prg files via File → Smart Attach, or drag and drop directly onto the emulator window.
