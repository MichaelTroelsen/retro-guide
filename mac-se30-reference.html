Motorola 68030 — Architecture Overview

The 68030 is a full 32-bit CISC processor introduced in 1987. It brought an on-chip MMU (Memory Management Unit), a 256-byte data cache, a 256-byte instruction cache, and a burst bus mode for faster memory access — making it a massive leap over the 68020 in the Mac II.

68030 Key Features

  • 32-bit data and address buses
  • On-chip paged MMU (no external 68851 needed)
  • 256-byte instruction cache (64 long-words)
  • 256-byte data cache (16 × 4-byte entries)
  • Dynamic bus sizing (8, 16, 32-bit)
  • Burst fill mode for cache lines
  • 14 addressing modes
  • Binary compatible with 68000/68010/68020
  • Up to 3.5 MIPS @ 16 MHz

Motorola 68882 FPU

  • IEEE 754 floating-point coprocessor
  • 80-bit extended precision internal registers
  • 8 × 80-bit floating-point data registers (FP0–FP7)
  • Transcendental functions in hardware (sin, cos, log…)
  • Communicates via coprocessor interface (CPI)
  • Significant benefit for CAD, spreadsheets, scientific apps
  • Not present on plain SE — SE/30 has it on-board

Frame Buffer Layout

The video frame buffer is 1-bit per pixel, stored row by row. Each horizontal line is 512 bits = 64 bytes. Pixel (0,0) is the top-left; bit 7 of the first byte is the leftmost pixel.

; Frame buffer base address FB_BASE EQU $FA700000 ; Calculate address of pixel (X, Y) ; Address = FB_BASE + (Y × 64) + (X ÷ 8) ; Bit within byte = 7 - (X MOD 8) ; Example: clear entire screen LEA $FA700000,A0 ; Point A0 at frame buffer MOVE.W #(21888/4)-1,D0 ; 5471 long-words to clear .loop CLR.L (A0)+ ; Clear 4 bytes (32 pixels) at once DBRA D0,.loop ; Repeat ; Set pixel at (X=10, Y=20) — set bit in frame buffer MOVE.L #20,D0 ; Y = 20 MULU.W #64,D0 ; Y × 64 bytes/row ADDI.L #1,D0 ; X=10 → byte offset = 10÷8 = 1 BSET #5,($FA700000,D0) ; bit = 7-(10 mod 8) = 5
QuickDraw — The Graphics API

All Mac software draws to the screen via QuickDraw, Apple's 2D graphics library baked into the ROM. Applications never write directly to the frame buffer; they use QuickDraw routines which manage the GrafPort, clipping regions, and coordinate transforms.

QuickDraw Core Concepts

  • GrafPort — Drawing destination (window or offscreen)
  • Pen — Current drawing mode, pattern, size
  • Pattern — 8×8 pixel fill patterns (stored in ROM)
  • Region — Arbitrary clipping/fill areas
  • BitMap — Rectangular 1-bit image data
  • Rect — Rectangle: top, left, bottom, right
  • Point — (h, v) coordinate pair

ADB is a slow (10 kbps) serial bus that replaced the older Macintosh keyboard/mouse interface. It supports up to 16 devices daisy-chained on a single 4-wire cable. Each device has a 4-bit address (0–15).

Sound — Apple Sound Chip (ASC)

The ASC provides 4-channel wavetable synthesis and PCM playback. It was Apple's first dedicated sound chip, replacing the PWM buzzer sound of earlier Macs. The SE/30's ASC drives a single internal speaker and the 3.5 mm mono output jack.

PDS → NuBus Adapter

Apple's IIsi NuBus adapter (also called the "SE/30 NuBus Adapter") lets you install two standard NuBus cards in the SE/30. This opens up the entire NuBus ecosystem including high-resolution colour video, DSP cards, and network adapters originally designed for the Mac II family.

⚠ Physical Clearance The NuBus adapter and cards protrude above the motherboard significantly. The SE/30's chassis is very tight — not all cards will physically fit. Cards must be low-profile or the lid may not close. Check compatibility lists before purchasing.

Key System Architecture

Mac Toolbox (ROM)

The Macintosh Toolbox is Apple's OS API — roughly 50,000 lines of 68000 assembly burned into 256 KB of ROM. Programs call ROM routines via A-line traps (instructions $A000–$AFFF).

  • QuickDraw — 2D graphics primitives
  • Event Manager — Mouse, keyboard, disk events
  • Window Manager — Overlapping windows
  • Menu Manager — Menus and menu bar
  • Dialog Manager — Dialogs and alerts
  • Memory Manager — Heap zones, handles
  • Resource Manager — Resource forks
  • File Manager — HFS file system access

A-Line Trap Mechanism

All Toolbox and OS calls use "A-line" traps — 68000 instructions with opcodes $Axxx that cause a privilege exception, handled by the ROM dispatcher.

; Trap format: $Axxx ; Toolbox: $A800–$AFFF ; OS: $A000–$A7FF _NewPtr ; = $A11E _DisposPtr ; = $A01F _SysBeep ; = $A9C8 _GetResource ; = $A9A0 _Gestalt ; = $A1AD
Gestalt Manager — System Feature Queries

The Gestalt Manager (introduced System 6.0.4) provides a standard API for querying hardware and software capabilities. Use it instead of hardcoded address sniffing.

; Gestalt call example (68030 assembly) ; D0 = selector code, A0 = pointer to result (OSType → long) MOVE.L #'proc',D0 ; gestaltProcessorType selector _Gestalt ; A-line trap — calls ROM ; A0 now points to response: gestalt68030 = 4 ; Common Gestalt selectors: 'proc' gestaltProcessorType — 4 = 68030 'fpu ' gestaltFPUType — 3 = 68882 'ram ' gestaltPhysicalRAMSize — bytes of RAM 'mach' gestaltMachineType — 9 = SE/30 'sysv' gestaltSystemVersion — e.g. $0755 = 7.5.5 'vm ' gestaltVMAttr — virtual memory status

⚠ ROM Requirements All SE/30 emulators require a ROM image dumped from a real SE/30 (or obtained through other means). The SE/30 ROM checksum is $36B8A5BF. Mini vMac's importer tool can extract it from a running physical Mac.

⚠ Known Failure Modes
Capacitor Plague — Priority #1 The SE/30 logic board uses cheap SMD electrolytic capacitors that universally fail after 25–35 years, leaking corrosive electrolyte that eats PCB traces and pads. Signs: no video, striped/corrupted display, random crashes, no boot chime. Repair requires full cap recap (47 caps, mostly 10µF 16V and 47µF 16V SMD).
PRAM Battery Corrosion The 3.6V lithium PRAM battery leaks and corrodes the logic board. Remove immediately if leaking. Replace with a compatible 1/2-AA 3.6V lithium cell with proper leads.
ℹ Simasimac / Checkerboard Display A checkerboard pattern on boot ("simasimac") indicates RAM issues — either bad SIMMs, faulty slots, or cap leak damage to RAM traces. Test with known-good SIMMs first.

💡 32-bit Clean ROM — Why It Matters The stock SE/30 ROM is "32-bit dirty" — it limits usable RAM to 8 MB without a software patch (MODE32). Installing a MACSIMM or GW4402B with a 32-bit clean ROM image eliminates this limitation entirely at the hardware level, and also speeds up boot time. It's one of the best single upgrades for an SE/30 running more than 8 MB RAM.
